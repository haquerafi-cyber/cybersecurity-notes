    # Basic
    
    ---
    
- Full Structure Format
    
    ```jsx
    nmap [Scan Type] [Port Option] [Detection Option] [Script Option] [Timing] [Output] [Target]
    
    এখন ছোট করে বুঝো 👇
    
    ১️⃣ Scan Type
    
    -sS → SYN Scan
    
    -sT → TCP Connect
    
    -sU → UDP
    
    ২️⃣ Port Option
    
    -p 80 → নির্দিষ্ট পোর্ট
    
    -p 80,443 → একাধিক পোর্ট
    
    -p- → সব পোর্ট
    
    ৩️⃣ Detection Option
    
    -sV → সার্ভিস ভার্সন
    
    -O → OS detect
    
    -A → সব ডিটেকশন একসাথে
    
    ৪️⃣ Script Option
    
    -sC → Default script
    
    --script vuln → Vulnerability check
    
    ৫️⃣ Timing
    
    -T0 → খুব ধীরে
    
    -T4 → দ্রুত
    
    ৬️⃣ Output
    
    -oN file.txt → টেক্সট ফাইল
    
    -oX file.xml → XML
    
    -oA name → সব ফরম্যাট
    
    🔹 বাস্তব উদাহরণ
    সাধারণ স্ক্যান
    nmap 192.168.1.1
    নির্দিষ্ট পোর্ট + ভার্সন
    nmap -p 80,443 -sV 192.168.1.1
    ফুল প্রফেশনাল ফরম্যাট
    nmap -sS -p- -sV -sC -O -T4 -oA report 192.168.1.1
    
    👉 মানে:
    চুপচাপ স্ক্যান করবে → সব পোর্ট → ভার্সন দেখবে → স্ক্রিপ্ট চালাবে → OS দেখবে → দ্রুত করবে → রিপোর্ট সেভ করবে
    ```
    

# ১. সাধারণ স্ক্যান (Basic Scans)

---

```jsx
নেটওয়ার্কের প্রাথমিক তথ্য জানার জন্য:

একটি নির্দিষ্ট আইপি স্ক্যান: nmap 192.168.1.1 (টার্গেট আইপি-র সাধারণ ১০০০টি পোর্ট স্ক্যান করে)

হোস্টনেম স্ক্যান: nmap google.com (ডোমেইন নাম দিয়ে স্ক্যান)

আইপি রেঞ্জ স্ক্যান: nmap 192.168.1.1-50 (১ থেকে ৫০ পর্যন্ত আইপি স্ক্যান)

পুরো সাবনেট স্ক্যান: nmap 192.168.1.0/24 (পুরো নেটওয়ার্কের সব ডিভাইস স্ক্যান)
```

# **২. হোস্ট ডিসকভারি (Host Discovery)**

```jsx
ডিভাইসটি চালু আছে কি না তা জানার জন্য (পোর্ট স্ক্যান ছাড়াই):

পিং স্ক্যান (Ping Scan): nmap -sn [target] (শুধু হোস্ট চালু আছে কি না দেখাবে)

নো পিং স্ক্যান (No Ping): nmap -Pn [target] (যদি টার্গেট পিং ব্লক করে রাখে, তবে এটি দিয়ে জোরপূর্বক স্ক্যান করা যায়)
```

# ৩. পোর্ট স্ক্যানিং টেকনিক (Port Scanning)

```jsx
স্টিলথ স্ক্যান (Stealth Scan): nmap -sS [target] (এটি খুব নিরব স্ক্যান, কানেকশন পুরোপুরি সম্পন্ন করে না)

টিসিপি কানেক্ট স্ক্যান: nmap -sT [target] (পুরো কানেকশন তৈরি করে, তাই স্ক্যান লগ-এ ধরা পড়ার সম্ভাবনা থাকে)

ইউডিপি স্ক্যান (UDP Scan): nmap -sU [target] (ইউডিপি পোর্টগুলো চেক করার জন্য)

নির্দিষ্ট পোর্ট স্ক্যান: nmap -p 80,443 [target] (শুধুমাত্র ৮০ এবং ৪৪৩ নম্বর পোর্ট স্ক্যান করবে)

সব পোর্ট স্ক্যান: nmap -p- [target] (১ থেকে ৬৫৫৩৫ পর্যন্ত সব পোর্ট চেক করবে)
```

# **৪. সার্ভিস ও ওএস ডিটেকশন (Service & OS Detection)**

```jsx
টার্গেটে কী সফটওয়্যার এবং অপারেটিং সিস্টেম চলছে তা জানার জন্য:

সার্ভিস ভার্সন: nmap -sV [target] (কী সার্ভিস চলছে এবং তার ভার্সন কত তা জানায়)

ওএস ডিটেকশন: nmap -O [target] (টার্গেট সিস্টেম উইন্ডোজ নাকি লিনাক্স তা বোঝার চেষ্টা করে)

অ্যাগ্রেসিভ স্ক্যান: nmap -A [target] (ওএস, ভার্সন, স্ক্রিপ্ট স্ক্যান এবং ট্রেসারুট একসাথে করে)
```

# **৫. এনম্যাপ স্ক্রিপ্টিং ইঞ্জিন (NSE Scripts)**

```jsx
অ্যাডভান্সড স্ক্যানিংয়ের জন্য এনম্যাপের বিল্ট-ইন স্ক্রিপ্ট:

ডিফল্ট স্ক্রিপ্ট: nmap -sC [target] (সাধারণ স্ক্রিপ্টগুলো রান করে)

ভালনারেবিলিটি স্ক্যান: nmap --script vuln [target] (টার্গেটে কোনো নিরাপত্তা দুর্বলতা আছে কি না দেখতে)

অথেনটিকেশন স্ক্যান: nmap --script auth [target] (পাসওয়ার্ড বা লগইন সংক্রান্ত দুর্বলতা চেক করতে)
```

# **৬. টাইমিং ও পারফরম্যান্স (Timing & Performance)**

```jsx
স্ক্যানের গতি নিয়ন্ত্রণ করার জন্য (T0 থেকে T5):

T0 / T1 (ধীরগতি): ফায়ারওয়াল ফাঁকি দেওয়ার জন্য খুব ধীরে ধীরে স্ক্যান করে।

T4 (দ্রুতগতি): সাধারণ এবং দ্রুত স্ক্যানের জন্য এটিই সেরা।

T5 (অত্যধিক দ্রুত): খুব দ্রুত স্ক্যান করে, কিন্তু নেটওয়ার্ক ক্র্যাশ করতে পারে।
```

# **৭. ফলাফল সেভ করা (Saving Results)**

```jsx
স্ক্যান রিপোর্ট ফাইল হিসেবে রাখার জন্য:

টেক্সট ফাইল: nmap -oN result.txt [target] (সাধারণ টেক্সট ফাইলে সেভ হবে)

এক্সএমএল ফাইল: nmap -oX result.xml [target] (প্রফেশনাল রিপোর্টের জন্য)

সব ফরম্যাট: nmap -oA filename [target] (একই সাথে টেক্সট, এক্সএমএল এবং গ্র্যাপেবল ফরম্যাটে সেভ হবে)
```

# **৮. ফায়ারওয়াল ফাঁকি দেয়া (Firewall Evasion)**

```jsx
ফায়ারওয়াল বাইপাস করার কিছু কৌশল:

প্যাকেট ফ্র্যাগমেন্ট: nmap -f [target] (প্যাকেটকে ছোট ছোট ভাগে ভাগ করে পাঠায় যাতে ফায়ারওয়াল ধরতে না পারে)

ডিকয় স্ক্যান (Decoy): nmap -D RND:5 [target] (নিজের আইপির সাথে আরও ৫টি ভুয়া আইপি মিশিয়ে স্ক্যান করে টার্গেটকে বিভ্রান্ত করে)
```
