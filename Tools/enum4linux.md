- প্রাথমিক প্রস্তুতি (Setup)
    
    প্রথমে তোমার পিসিতে Git ইনস্টল থাকতে হবে এবং একটি GitHub অ্যাকাউন্ট থাকতে হবে। টার্মিনাল বা কমান্ড প্রম্পটে গিয়ে নিজেকে পরিচয় করিয়ে দিতে এই কমান্ডগুলো একবার দিতে হয়:
    
    ```jsx
    git config --global user.name "Your Name"
    
    git config --global user.email "your@email.com"
    ```
    

# **জরুরি গিট কমান্ডসমূহ (The Essentials)**

---

```jsx
 কমান্ড	                                   কাজ কি?
git init	                    নতুন একটি গিট রিপোজিটরি (ফোল্ডার) শুরু করা।

git clone [URL]	              গিটহাব থেকে কোনো প্রোজেক্ট নিজের পিসিতে কপি করে আনা।

git status                   	কোন ফাইলগুলো পরিবর্তন হয়েছে বা সেভ হয়নি তা দেখা।

git add .	                    সব পরিবর্তনগুলোকে সেভ করার জন্য লাইনে দাঁড় করানো (Staging)।

git commit -m "বার্তা"	          পরিবর্তনগুলোকে একটি নাম দিয়ে স্থায়ীভাবে সেভ করা।

git push origin main	        তোমার পিসির কোড গিটহাব সার্ভারে পাঠিয়ে দেওয়া।

git pull	                    গিটহাব থেকে লেটেস্ট কোড নিজের পিসিতে নামিয়ে আনা।
```

# কাজের ফ্লো (Workflow) - তুমি যেভাবে কাজ করবে

```jsx
তুমি যখন কোনো প্রোজেক্টে কাজ করবে, সাধারণত এই চক্রটি ফলো করবে:

পরিবর্তন করো: তোমার কোড এডিটরে  কাজ করো।

চেক করো: git status দিয়ে দেখো কি কি পরিবর্তন হলো।

স্টেজ করো: git add . দিয়ে ফাইলগুলো রেডি করো।

সেভ করো: git commit -m "Added a new feature" দিয়ে একটি স্ন্যাপশট নাও।

আপলোড করো: git push দিয়ে দুনিয়াকে দেখানোর জন্য গিটহাবে পাঠিয়ে দাও।
```

# ব্রাঞ্চিং (Branching) - প্রফেশনালদের সিক্রেট

---

```jsx
মনে করো তোমার মেইন কোড ঠিক আছে, কিন্তু তুমি নতুন একটা এক্সপেরিমেন্ট করতে চাও। তখন তুমি মেইন কোড নষ্ট না করে একটি Branch তৈরি করবে।

git branch [name] : নতুন ব্রাঞ্চ তৈরি করা।

git checkout [name] : ওই ব্রাঞ্চে চলে যাওয়া।

git merge [name] : কাজ শেষ হলে মেইন কোডের সাথে যোগ করে দেওয়া।
```
